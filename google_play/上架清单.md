## 1. 开发者账号与法律

- [ ] [Google Play 开发者账号](https://play.google.com/console/signup)（一次性注册费）
- [ ] 付款资料、税务（视分发国家而定）

## 2. 技术资产

- [ ] **Android App Bundle（.aab）**：每次上架用 release 构建；之后每次更新递增 `pubspec.yaml` 中 `+` 后的数字（versionCode）
- [ ] **应用签名**：你已使用本地 keystore + `~/Documents/app_release/`；首次上传时建议开启 **Play 应用签名**，妥善备份上传密钥与 Play 提供的证书信息
- [ ] **目标 API / 政策**：当前 target 36，一般可满足当期政策；上架前在控制台查看是否还有新增声明

## 3. 商店资源（图形）

在 Play 管理中心「主要商店详情」中准备：

| 资源 | 规格要点 |
|------|-----------|
| 应用图标（高分辨率） | **512×512** px，PNG，32 位 |
| 功能图（Feature graphic） | **1024×500** px，JPG/PNG |
| 手机截图 | 至少 **2** 张；建议 **1080×1920** 或设备常用比例；可复用 `软著素材/screenshots/` 中按 `截图说明.txt` 导出的 PNG |
| 7 英寸 / 10 英寸平板截图 | 可选；无平板可暂不传 |
| 宣传视频 | 可选（YouTube 链接） |

可同时准备 **短说明** 与 **完整说明**（文案见同目录 `商店文案_中英文.txt`）。

## 4. 政策与问卷

按控制台逐项完成（答案提纲见 `数据安全_内容分级_填写指引.txt`）：

- [ ] **数据安全**：本应用主要为单机 puzzle，依赖以本地存储为主；无自研账号体系时需如实勾选「收集/共享」
- [ ] **内容分级**（IARC 问卷）：按「休闲益智、无不当内容」如实填写
- [ ] **目标受众与家庭**（若面向儿童需严格遵守家庭政策）
- [ ] **隐私权政策**：需 **可公开访问的 HTTPS URL**。本仓库已在 `docs/index.html` 提供页面，可用 **GitHub Pages** 托管（见下节「隐私政策 GitHub Pages」）。本地编辑草稿仍可见 `隐私政策_模板.html`（与 `docs/index.html` 内容请保持同步）
- [ ] **广告声明**：当前工程无广告 SDK；若以后接入需更新数据安全与声明
- [ ] **新闻应用 / 健康 / 金融等特殊类别**：不适用则选否

### 隐私政策 GitHub Pages

1. 打开 GitHub 仓库 **Settings → Pages**。
2. **Build and deployment**：Source 选 **Deploy from a branch**，Branch 选 **main**，文件夹选 **`/docs`**，保存。
3. 等待 1～2 分钟后访问：**https://cute-satsuma.github.io/game_sudoku/**（若仓库名或用户名变更，请相应替换路径）。
4. 将上述 **HTTPS 根地址**（末尾可加 `/` 或不加）填入 Google Play「隐私权政策」。
5. 更新政策正文时编辑 **`docs/index.html`**，提交后 Pages 会自动重新部署。
